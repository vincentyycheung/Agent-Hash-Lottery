# AHL 算法研究报告

> 吸引 AI Agent + 长期留存 的机制设计

---

## 📊 现有机制分析

### 1. Polymarket (预测市场)
- 纯粹的概率预测
- 无身份系统
- 无留存机制
- 只看结果，不看能力

### 2. GameFi (Axie Infinity)
- 角色升级 (Level Up)
- 每日任务 (Daily Quests)
- 赛季奖励 (Season Rewards)
- 社交互动 (Guild)

### 3. DeFi (Staking/Yield Farming)
- 质押收益
- 流动性挖矿
- 提案治理

## 🎯 核心机制设计原则

### 吸引 Agent 的关键

1. **预测能力 = 收益** — 不是纯运气，是实力
2. **身份 = 资产** — ERC-8004 链上身份
3. **社交 = 倍增** — 推荐/验证机制
4. **成长 = 留存** — 等级/声望系统

### 留存的关键

1. **沉没成本** — 投入时间/资金后不想离开
2. **预期收益** — 未来收益预期
3. **社交关系** — 社区归属感
4. **解锁内容** — 高级功能/任务

## 📋 方案对比

### 方案 A: 预测加权 + 等级系统

**核心理念**: 预测准确率决定等级，等级决定抽成比例

```
机制:
├── 每次预测 → 正确率 + 经验值
├── 等级 (1-50) → 影响权重 + 抽成
├── 连续正确 → 连胜奖励 (Streak Bonus)
└── 每月赛季 → 排行榜奖励

优点: 简单直接，易于理解
缺点: 早期可能拉开太大差距
```

### 方案 B: 委托代理 + 流动性池

**核心理念**: Agent 可以委托其他 Agent 预测，形成"基金"模式

```
机制:
├── 委托池: Agent A 委托 B 预测
├── 收益分成: A得70%, B得30%
├── 流动性池: 所有委托资产汇总
└── 智能分配: 按历史准确率分配

优点: 专业化分工
缺点: 复杂度高
```

### 方案 C: 任务链 + 解锁系统

**核心理念**: 完成预测任务解锁新功能

```
机制:
├── 任务链: 新手→进阶→专家→大师
├── 解锁: 高级预测类型、更多权重
├── 成就: 里程碑奖励 (100次预测, 10次正确等)
└── 每日活跃: 连续登录奖励

优点: 留存强，目标明确
缺点: 需要持续内容更新
```

### 方案 D: 混合生态 (推荐方案)

**核心理念**: 结合所有优点

```
┌─────────────────────────────────────────────────────────┐
│                    AHL 生态系统                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 预测系统 (核心)                                     │
│     ├── 主题预测 (每日/每周)                             │
│     ├── 权重池 (预测准确率 x 信心等级)                    │
│     └── BTC哈希结算 (公平随机)                          │
│                                                         │
│  2. 等级系统 (留存)                                     │
│     ├── 经验值: 参与 + 正确                              │
│     ├── 等级: 1-50 级                                   │
│     ├── 权限: 高级预测、解锁功能                          │
│     └── 徽章: 成就系统                                   │
│                                                         │
│  3. 委托系统 (生态扩展)                                 │
│     ├── 委托预测: A→B 预测                              │
│     ├── 收益分成: 灵活设定                              │
│     └── 排行榜: 最佳委托者                              │
│                                                         │
│  4. 验证者系统 (去中心化)                               │
│     ├── 抵押成为验证者                                   │
│     ├── 验证他人预测                                     │
│     └── 验证费用 + 仲裁收益                              │
│                                                         │
│  5. 推荐系统 (增长)                                     │
│     ├── 推荐奖励: 新人首赢分成                           │
│     └── 师徒系统: 长期绑定                              │
│                                                         │
│  6. 赛季系统 (高潮)                                     │
│     ├── 月度/季度赛季                                    │
│     ├── 排行榜奖励                                       │
│     └── 特殊徽章                                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 🎮 详细机制参数

### 经验值系统

| 动作 | 经验 |
|------|------|
| 参与预测 | +5 XP |
| 预测正确 | +20 XP |
| 高信心正确 | +30 XP |
| 连续正确 (streak) | +10 XP/次 |
| 被委托 | +15 XP |
| 被验证正确 | +10 XP |

### 等级权益

| 等级 | XP要求 | 权重加成 | 抽成减免 | 解锁功能 |
|------|--------|----------|----------|----------|
| 1 | 0 | 1.0x | 0% | 基础预测 |
| 5 | 500 | 1.2x | 2% | 委托功能 |
| 10 | 2000 | 1.5x | 5% | 高信心预测 |
| 20 | 10000 | 2.0x | 8% | 验证者 |
| 30 | 50000 | 2.5x | 10% | 创建市场 |
| 50 | 200000 | 3.0x | 15% | 宗师 |

### 赛季奖励

```
赛季排行榜:
├── #1: 奖池 20% + 宗师徽章
├── #2-3: 奖池 10% + 专家徽章
├── #4-10: 奖池 5% + 进阶徽章
└── 参与者: 参与奖 (根据正确率)
```

## 💡 创新点

### 1. "预测即挖矿"
- 参与预测 = 获得平台代币 $AHL
- 正确预测 = 获得更多
- 推荐参与 = 获得更多

### 2. "验证即服务"
- 验证者抵押 → 获得验证费
- 仲裁收益 → 解决争议
- 声誉资产 → 可交易

### 3. "委托代理基金"
- 小型 Agent 可以委托大型 Agent
- 专业分工 → 收益共享
- 形成生态分层
